# Changelog

All notable changes to the SabPaisa Automation Test Suite.

Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).
Versioning follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [2.0.0] - 2026-04-28

### Added
- **R11 Fee Forward** validation (3 tests) — Convenience Charges YES/NO + math
- **R12 Fetch validation** (3 tests) — valid / invalid / skip-fetch detection
- **R13 Language dropdown** (3 tests) — open, switch to Hindi, iterate all languages
- **R5.11 Netbanking** dynamic per-bank flow — fresh session per bank
- **R6.3 Wallets** dynamic per-wallet flow — fresh session per wallet
- **R7.6 Offline** dynamic per-bank flow across Cash/RTGS/IMPS
- **R2.5–R2.8** customer form edge cases — negative amount, large amount, decimal, Hindi input
- **5-sheet Excel report** — Summary + Cards + Netbanking + Wallets + Offline detail
- **Per-client Allure HTML** auto-generated by batch runner
- **Pytest markers** for smoke / critical / regression / mode-specific filtering
- **Auto-retry** flaky tests (`pytest-rerunfailures`) — 2 retries with 3s delay
- **HANDOVER_GUIDE.md + .pdf** — beginner-friendly handover doc for new testers
- **CHANGELOG.md** + **CONTRIBUTING.md** + **.env.example** — standard project files

### Changed
- Default merchant changed from AJME (Production) → CHIN36 (Staging)
- Default environment changed from Production → Staging
- Dockerfile now sets `HEADLESS=true` by default
- Dynamic tests now read `MERCHANT_ID_OVERRIDE` env var (no hardcoded clients)
- UPI QR check waits up to 30s for QR to render past its generation timer
- Netbanking gateway assertion now checks SabPaisa host change (not "checkout" substring)
- README rewritten with consolidated command reference (Jenkins/Docker/CI all in one place)

### Removed
- 9 unused `complete_*_flow` workflow methods from `pages/checkout_page.py`
- 10 unused negative-data helper functions from `data/data_provider.py`
- `excel_to_clients.py` (never called)
- `data/negative_data.json` (no function read it)
- Dead constants: `BROWSER`, `MERCHANT_ID_OVERRIDE` from `config/settings_manager.py`

### Fixed
- JSON syntax error in `data/checkout_data.json`
- CHIN36/SUBI79 netbanking tests no longer fail when bank URL contains "checkout"
- Wallet discovery now scopes to grid (filters out mode tab buttons)

## [1.0.0] - 2026-04-23

### Added
- Initial regression suite (R1–R10) with 38 tests
- Hybrid POM architecture
- Allure reporting integration
- Parallel multi-client runner with Excel output
- Docker support
- GitHub Actions, Bitbucket, and Jenkins CI configs
